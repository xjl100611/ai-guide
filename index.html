💡 面试高频问答（35 题）

▶ Q1：Transformer 的 Self-Attention 是怎么计算的？为什么需要除以 √dₖ？

计算流程：

  1. 输入 X 通过 Wq、Wk、Wv 投影得到 Q、K、V
  2. 注意力分数：Score = Q·Kᵀ
  3. 缩放：Score / √dₖ
  4. Softmax 归一化 → 注意力权重
  5. 加权求和：Output = 权重 · V

除以 √dₖ 的原因：当 dₖ 很大时，点积值会很大导致 Softmax 进入梯度极小区。除以 √dₖ 将方差控制为 1，保持梯度稳定。

📺 B站：李沐精读 Attention 论文（逐行代码讲解）

▶ Q2：Multi-Head Attention 为什么比单头好？

不同 Head 关注不同的表示子空间——有的关注位置关系，有的关注语法结构，有的关注语义相似度。多个 Head 并行计算后拼接，让模型同时从多个角度理解输入。

▶ Q3：LayerNorm 和 BatchNorm 的区别？为什么 Transformer 用 LN？
  • BN：在 batch 维度归一化（同一特征在不同样本上）
  • LN：在 feature 维度归一化（同一样本的不同特征上）

Transformer 用 LN 的原因：序列长度可变（BN 对 padding 敏感）；LN 不依赖 batch size（训练推理一致）；序列模型中 BN 统计量不稳定。

▶ Q4：位置编码：Sinusoidal、可学习、RoPE 各有什么优劣？
  • Sinusoidal：固定正余弦函数。优势：可外推到训练时未见的长度；劣势：不可学习
  • 可学习：作为参数训练。优势：灵活适应数据；劣势：无法外推
  • RoPE（旋转位置编码）：2026 年主流。通过旋转矩阵编码相对位置，兼具外推性和可学习性。Qwen/DeepSeek/ChatGLM 均使用
▶ Q5：Pre-Norm vs Post-Norm 的区别？
  • Post-Norm（原论文）：LayerNorm(x + Sublayer(x))
  • Pre-Norm（现在主流）：x + Sublayer(LayerNorm(x))

Pre-Norm 优势：训练更稳定，不需要 warmup；梯度流动更顺畅。几乎所有现代大模型都用 Pre-Norm。

▶ Q6：全量微调、LoRA、QLoRA 的区别？什么时候用哪种？
方法原理显存 (7B)效果适用场景
全量微调更新所有参数~56GB最好数据多、算力充足
LoRA低秩矩阵分解，只训练旁路矩阵~16GB接近全量资源有限，通用首选
QLoRA4-bit 量化 + LoRA~8GB略低于 LoRA消费级显卡

选择原则：有 A100 → 全量微调；有 RTX 4090 → LoRA；笔记本显卡 → QLoRA。

🔧 B站：LLaMA-Factory 微调教程

▶ Q7：LoRA 的 rank (r) 和 alpha 参数怎么选？
  • r（秩）：通常 8-64。复杂任务（数学推理）r 越大；简单任务（情感分类）r=8 够用。r 翻倍 = 可训练参数翻倍
  • alpha：缩放因子，通常设为 r 的 2 倍（r=16, alpha=32）。alpha 越大，LoRA 对原始权重的修改幅度越大
▶ Q8：什么是灾难性遗忘？微调时如何缓解？

灾难性遗忘 = 模型微调后新任务变好但原有能力退化。

缓解方法：LoRA 低秩微调（只改少量参数）；混合数据训练（微调数据+原始预训练数据混合）；渐进式微调；定期在通用 benchmark 上评估。

▶ Q9：RLHF 和 DPO 的区别？为什么 DPO 更快？
  • RLHF：训练 Reward Model → PPO 强化学习。三阶段，复杂且不稳定
  • DPO（Direct Preference Optimization）：直接从偏好对中学习，合并 RM 和 Policy 优化为一步。训练稳定且快

DPO 是 2025-2026 年对齐方法的主流选择。

▶ Q10：RAG 的完整流程是怎样的？

离线索引：文档加载 → 分块 (Chunking) → Embedding → 存入向量库

在线检索：用户查询 → Query 改写/扩展 → Embedding → 向量检索 (Top-K) → 重排序 (Reranker)

生成：检索到的文档 + 原始查询 → 构造 Prompt → LLM 生成回答（含引用）

🔧 B站：RAG 全流程实战教程

▶ Q11：文档分块 (Chunking) 有哪些策略？
  • 固定大小分块（512 tokens，重叠 128）：最简单，适合通用文档
  • 语义分块（基于 Embedding 相似度切分）：段落边界模糊时用
  • 递归分块（按段落→句子→词逐级切分）：结构化文本
  • 文档结构分块（按 Markdown 标题层级）：技术文档

2026 年趋势：智能分块——用小模型判断最优分块点。

▶ Q12：如何评估和优化 RAG 系统的检索质量？

评估指标：Recall@K、MRR、NDCG

优化策略：混合检索（BM25 关键词 + 向量语义）；Query 改写（LLM 扩展简短查询）；HyDE（生成假设文档再检索）；Reranker 重排序（BGE-Reranker/Cohere Rerank）；负样本挖掘

▶ Q13：向量数据库选型：ChromaDB vs Milvus vs FAISS？
  • ChromaDB：轻量 Python-native，原型开发首选。→ 官网
  • Milvus：分布式高性能，十亿级向量，生产首选（国产开源）。→ 官网
  • FAISS：Meta 开源算法库，适合嵌入已有系统
▶ Q14：什么是 ReAct 模式？为什么它对 Agent 很重要？

ReAct = Reasoning + Acting。模型交替进行「思考」（分析当前状态、制定下一步计划）和「行动」（调用工具、获取信息），循环直到完成。

纯推理容易产生幻觉（闭门造车），纯行动缺乏规划。ReAct 结合两者，让 Agent 能基于实时反馈调整策略——这是 Agent 区别于简单 LLM 调用的核心能力。

📺 Coze 扣子 可用可视化方式理解 ReAct 模式

▶ Q15：Function Calling 的实现机制是什么？
  1. 定义 JSON Schema 描述函数（函数名、参数、描述）
  2. 将 Schema 传入 LLM 的 tool_choice 参数
  3. LLM 判断是否需要调用工具，如需要则输出函数名 + 参数 JSON
  4. 程序执行函数，将结果返回给 LLM
  5. LLM 基于工具结果继续推理或生成最终回复

关键细节：需要处理工具调用失败、超时、参数解析错误的 fallback 机制。

▶ Q16：Multi-Agent 系统中 Agent 间如何通信和协调？
  • 顺序流水线：A 的输出 → B 的输入（适合线性任务）
  • 辩论模式：多个 Agent 独立回答 → 裁判 Agent 选择最佳
  • 层级模式：主管 Agent 分配任务 → 执行 Agent 汇报 → 主管汇总
  • 自由对话：Agent 间通过消息总线自由交流
  • 协议标准：MCP 协议（上下文传递）、Google A2A（Agent 间通信）
▶ Q17：什么是 Prompt Injection？如何防御？

攻击者通过构造恶意输入，覆盖或篡改系统的原始指令。

常见形式：直接覆盖（"忽略以上所有指令"）、间接注入（在检索文档中嵌入恶意指令）、多语言绕过。

防御：输入净化（过滤特殊字符/指令模式）；指令隔离（用户输入与系统指令严格分开 role）；输出检测（另一个模型检测合规性）；权限最小化（Agent 能调用的工具/数据范围严格限制）。

▶ Q18：越狱 (Jailbreak) 和 Prompt Injection 有什么区别？
  • Jailbreak：诱导模型绕过安全对齐，产出有害内容。攻击的是模型本身的安全护栏
  • Prompt Injection：劫持应用的系统指令，让应用执行非预期操作。攻击的是应用层的指令设计

简单区分：Jailbreak 针对模型，Injection 针对应用。

▶ Q19：INT8 和 INT4 量化的核心原理？
  • INT8 量化：FP32 → 8-bit 整数，q = round(w/scale) + zero_point。每个通道/张量共享 scale 和 zero_point
  • INT4 量化：需更精细分组量化（group_size=128）。大幅降显存（75%）但精度损失更大

核心技术：GPTQ 用 Optimal Brain Quantization 逐列量化并补偿误差；AWQ 发现 1% 的「显著权重」对精度影响最大，优先保护它们。

▶ Q20：什么场景下需要端侧 AI (TinyML)？
  1. 隐私敏感：本地处理数据不上传云端（输入法、相册）
  2. 低延迟要求：自动驾驶、实时翻译不能容忍网络延迟
  3. 离线场景：没有网络的场景（野外巡检、飞行模式）
  4. 成本控制：减少 API 调用费用
  5. 可穿戴/IoT：设备算力有限但需要 AI 能力
▶ Q21：你做的 AI 项目中遇到的最大技术挑战是什么？怎么解决的？

用 STAR 框架：

  • Situation：项目背景和目标
  • Task：你的具体任务
  • Action：技术方案（为什么选这个，还有哪些备选）
  • Result：量化结果（延迟降低 X%、准确率提升 Y%）
▶ Q22：如果业务方要求一周上线但你评估需三周，怎么沟通？

先理解真正的业务需求（往往不是"AI 功能"而是"解决什么问题"），然后提出分阶段交付方案：第 1 周 MVP（用现有 API 拼一个原型）→ 第 2 周优化核心体验 → 第 3 周打磨上线。在第一次沟通时就亮出替代方案，而不是只说"做不完"。

▶ Q23：你怎么看待 AI 伦理和合规？
  • 数据合规（隐私保护、用户授权）
  • 内容安全（不生成有害/违法内容）
  • 公平性（模型对不同人群的识别率是否一致）
  • 可解释性（用户有权知道 AI 决策的依据）
  • 人类把关（高风险场景必须有人工审核环节）

▶ Q31：为什么国产模型都用 Decoder-only 架构？

Decoder-only 在 scaling law 下表现稳定，训练效率高，社区生态（Qwen/ChatGLM/Llama 系列）都是这个架构，方便对齐和复用。国内 Qwen、DeepSeek、ChatGLM 全是 Decoder-only。

▶ Q32：华为昇腾和英伟达 GPU 在 AI 训练上的差异？如何适配？（投华为系必考）

  • 混合精度训练需适配昇腾的融合算子
  • 面试加分项：简历中体现昇腾适配经验直接加分
